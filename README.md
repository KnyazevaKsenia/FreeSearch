# FitFlow

## 1. Product Vision

**FitFlow** — это сервис, который помогает фрилансерам быстрее находить релевантные заказы за счёт интеллектуальной фильтрации и агрегации предложений с разных платформ.

### Проблема
Встроенные фильтры фриланс-площадок не учитывают реальный уровень навыков, опыт, специализацию и предпочтения пользователя. В результате фрилансеры тратят значительное количество времени на ручной поиск и просмотр нерелевантных заказов.

### Решение
FreeSearch предлагает единоразовое заполнение анкеты с подробным описанием навыков, стека технологий, опыта и предпочтений. На основе этих данных система автоматически:
- агрегирует заказы с разных платформ,
- анализирует их соответствие профилю пользователя,
- отправляет только релевантные предложения в Telegram сразу после публикации.

### Целевая аудитория
- Фрилансеры (разработчики, дизайнеры, маркетологи, аналитики и др.)
- Начинающие специалисты, ищущие первые проекты
- Опытные специалисты, ценящие экономию времени

### Ценность продукта
- Экономия времени
- Повышение релевантности заказов
- Быстрое реагирование на новые предложения
- Централизация поиска из разных источников

## 2. Functional Requirements

### 2.1 Регистрация и профиль
- Пользователь аворизуется через Telegram.
- Пользователь может заполнить анкету с описанием:
  - навыков
  - технологий и инструментов
  - уровня опыта
  - желаемой ставки
  - формата работы (удалённо, частичная занятость и т.д.)
  - языка проекта
- Пользователь может редактировать профиль в любое время.

### 2.2 Сбор заказов
- Система автоматически собирает заказы с нескольких фриланс-платформ.
- Сбор данных происходит с заданной периодичностью (например, каждые 1-2 минуты).
- Система сохраняет заказы в базе данных.

### 2.3 Фильтрация и матчинг
- Система анализирует описание заказа.
- Система сопоставляет заказ с профилем пользователя.
- Используется алгоритм оценки релевантности.
- Заказ отправляется пользователю, если он превышает заданный порог соответствия.

### 2.4 Уведомления
- Подходящие заказы отправляются пользователю в Telegram.
- Уведомление содержит:
  - название проекта
  - краткое описание
  - бюджет
  - ссылку на оригинальную платформу
- Пользователь может настроить частоту уведомлений.

### 2.5 Настройки
- Пользователь может:
  - включать/выключать уведомления
  - задавать минимальный бюджет
  - выбирать категории проектов
  - настраивать порог релевантности

### 2.6 Автогенерация сопроводительного письма
- Система автоматически генерирует персонализированное сопроводительное письмо для каждого релевантного заказа.
- Генерация письма основывается на:
  - профиле пользователя
  - навыках и опыте
  - тексте конкретного заказа
- Пользователь может:
  - редактировать сгенерированное письмо перед отправкой
  - выбрать стиль письма (формальный, краткий, экспертный и т.д.)
  - сохранить собственные шаблоны
- Система может учитывать предыдущие успешные отклики для улучшения качества генерации.

### 2.7 Администрирование
- Администратор может добавлять новые платформы.
- Администратор может управлять пользователями.
- Администратор может просматривать статистику использования.

## 3. Non-Functional Requirements

### 3.1 Производительность
- Уведомления за < 2 минуты с момента публикации заказа.
- Система должна поддерживать масштабирование при росте числа пользователей.
- Обработка до 1000 заказов/мин в пике.

### 3.2 Безопасность
- Данные пользователей должны храниться в зашифрованном виде.
- Доступ к административной панели ограничен ролями.

### 3.3 Масштабируемость
- Архитектура должна позволять добавлять новые источники заказов без значительной переработки системы.
- Система должна поддерживать горизонтальное масштабирование.

### 3.4 Юзабилити
- Интерфейс должен быть интуитивно понятным.
- Время заполнения анкеты занимает не более 2-3 минут.
- Telegram-уведомления должны быть краткими и информативными.

- Встроенная аналитика по рынку заказов.
